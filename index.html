Project 1: Helpdesk Ticket Resolution Case Study: Network Connectivity Issue

Objective: To demonstrate the structured troubleshooting approach and resolution techniques for resolving complex helpdesk tickets in a professional IT environment.

Scenario

Ticket Details:
Issue: Remote employee unable to connect to the company's VPN.
Priority: High (affecting work productivity).
Reported Symptoms:
• The VPN client fails to establish a connection.
• Error message: "Unable to establish a secure connection."
• Other internet services working as expected.

Impact:
Employee cannot access internal company resources critical for their role.

Assigned Technician: Leo


Troubleshooting Process

Step 1: Initial Diagnosis

1. Contact the User:
• Confirm the error and gather additional details (when the issue started, recent system changes, etc.).
• Verify the user's environment:
  - Device: Windows 11 laptop
  - Internet: Stable home Wi-Fi

2. Reproduce the Issue:
• Ask the user to attempt reconnecting to the VPN.
• Confirm the exact error message.

3. Assess Common Issues:
• Check for basic problems: incorrect credentials, outdated software, or firewall interference.

Step 2: Identify Root Cause

1. Check System Logs:
• Ask the user to open the VPN client log files and share details.
• Logs showed error ERR-101: Unable to negotiate encryption protocol.

2. Network Diagnostics:
• Ping the VPN gateway to confirm accessibility (ping vpn.companydomain.com).
  - Response: "Request timed out."
• Verify other online services like web browsing.

3. Remote Inspection:
• Use remote desktop software (e.g., TeamViewer or AnyDesk) to access the user's machine.
• Verify VPN settings, system date/time, and installed software version.

4. Firewall and Antivirus Check:
• Found the user's antivirus (Norton) blocking the VPN application.

Step 3: Implement Solutions

1. Update VPN Client:
• Installed the latest version of the VPN client compatible with the company’s server.

1. Check System Logs: Configure Antivirus:
• Added the VPN client as an exception in the Norton firewall and antivirus settings.

1. Check System Logs: Reconnect to VPN:
• Restarted the user's device to apply the changes.
• Attempted to connect, but the error persisted.

1. Check System Logs: Network Gateway Verification:
• Verified that the VPN gateway was operational by checking with the network administrator.
• Discovered the VPN server IP had recently changed due to a server migration, but the user's profile was still using the old IP.

Step 4: Final Resolution

1. Update VPN Server Address:
• Edited the user's VPN configuration to reflect the new gateway address.

2. Reconnect to VPN:
• Successfully established a secure connection.

3. Test Connectivity:
• Verified access to internal company resources.
• Confirmed the user could resume their work without issues.

Tools Used
• Diagnostic Tools: Ping, tracert, and VPN log analysis.
• Remote Access Software: TeamViewer.
• Antivirus Configuration: Norton Security Console.
• Communication: Teams for real-time guidance.

Best Practices Followed
• Systematic Approach: Followed a structured process to isolate the issue step by step.
• Clear Communication: Maintained consistent updates with the user.
• Documentation: Recorded all steps and findings for knowledge base updates.
• Preventive Measures: Created an internal memo to alert all remote users about updating their VPN profiles.

Project 2: Hardware & Software Setup Guide: Onboarding a New Employee's Workstation

Objective: This project outlines a professional and secure process for setting up a new employee’s workstation, covering hardware configuration, software installation, and data security measures. The guide is designed to highlight your technical expertise and attention to detail.

Scenario

Task: Prepare a workstation for a new employee joining the IT department.
Requirements:
• Hardware: Laptop (Dell Latitude 5430)
• Software: Windows 11 Pro, Microsoft Office 365, Adobe Creative Cloud, and Slack.
• Security: Enable encryption, configure user permissions, and install endpoint protection.

Step-by-Step Setup Process

Phase 1: Hardware Setup

1. Unbox and Inspect Equipment:
• Verify that all components are present and undamaged (laptop, charger, docking station, peripherals).

2. Physical Setup:
• Connect the laptop to the docking station.
• Attach external peripherals (monitor, keyboard, mouse, and Ethernet cable).

3. Initial Boot and BIOS Configuration:
• Power on the laptop.
• Access BIOS (press F2 or DEL during boot).
• Configure boot order and enable Secure Boot.
• Enable TPM 2.0 for encryption compatibility.

Phase 2: Software Installation

1. Operating System Installation:
• Preinstalled Windows 11 Pro is verified during startup.
• Apply all pending Windows Updates (Settings > Update & Security > Check for Updates).

2. Install Required Software:
• Microsoft Office 365:
  - Login to the company’s Office 365 portal.
  - Download and install Office apps.
• Adobe Creative Cloud:
  - Download and install using the company-provided license key.
• Slack:
  - Install the desktop application from the official website.

3. Additional Configuration:
• Set up OneDrive for automatic backup of the Documents folder.
• Configure email client (Microsoft Outlook) with company credentials.

Phase 3: Security Setup

1. Data Encryption:
• Enable BitLocker on the primary drive:
  - Go to Control Panel > BitLocker Drive Encryption.
  - Select "Turn on BitLocker" and save the recovery key to a secure location.

2. Endpoint Protection:
• Install and configure antivirus software (e.g., Windows Defender or Sophos Endpoint).
• Run a full system scan to confirm the system is secure.

3. User Account Permissions:
• Set up a Standard User Account for daily use to limit admin privileges:
  - Control Panel > User Accounts > Add Account.
  - Assign admin rights to the IT team account only.

4. Firewall and Network Configuration:
• Enable Windows Firewall.
• Connect to the company’s VPN and test connectivity to internal resources.

Phase 4: Testing and Handover

1. Test Functionality:
• Verify hardware components (monitor, keyboard, mouse).
• Confirm software launches without errors.
• Test connectivity to shared drives and company intranet.

2. Documentation and Handover:
• Record system details (serial number, hostname, IP address).
• Provide the new employee with a Quick Start Guide and credentials.
• Schedule a follow-up for any required adjustments.

Security Considerations

1. Data Protection:
• BitLocker ensures data is encrypted, preventing unauthorized access if the device is lost or stolen.
• OneDrive backup mitigates the risk of data loss.

2. Access Control:
• Assigning standard user roles minimizes the risk of unauthorized software installations.
• All sensitive configurations require admin approval.

3. Network Security:
• VPN encrypts communication between the device and the corporate network.
• Firewalls block unauthorized inbound and outbound traffic.
